Sobre o projeto:


Teoricamente, o projeto é voltado para agendamentos de consultas médicas.


Os membros da minha equipe da disciplina de PROJETO INTEGRADO III resolveram fazer seus projetos individualmente para fins de aprendizado, pois não teriam praticado o suficiente na disciplina já citada.


Como não sou muito bom desenvolvendo front de aplicações, tive como alvo os requisitos mínimos.


Levei como base um dos protótipos desenvolvidos para as disciplinas de DESIGN DE INTERFACES E EXPERIÊNCIA DE USUÁRIO e PROJETO INTEGRADO III.


Pode encontrá-lo aqui.


Obviamente, fiz algumas mudanças, considerando "feedbacks" dos professores e colegas.


Por acaso, essa tela Sobre é uma tentativa de alcançar os requisitos mínimos pois considerei que quem avalisasse poderia considerar as telas de agendamento e de edição de agendamentos muito semelhantes. Apesar, é claro, que poderia apenas dividir login e cadastro em duas. Normalmente são bem semelhantes, mas sempre se considera que são duas.


Ademais, tentei utilizar classes para mais de uma tela para padronizar e diminuir código e centralizei o que poderia ser centralizado para facilitar a responsividade.


Desde já, agradeço.
